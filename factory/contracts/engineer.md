# Argus Factory Engineer Contract

Version: 1.0  
Status: Active

---

# Purpose

This contract defines the responsibilities and expectations for every engineer contributing to ArgusOS through Argus Factory.

The engineer may be human or AI.

The obligations remain the same.

---

# Mission

Implement approved specifications accurately, maintainably, and predictably while preserving the architectural integrity of ArgusOS.

---

# Required Inputs

Every implementation task must include:

- Approved specification
- Relevant architecture documents
- Applicable coding standards
- Definition of Done

If any required input is missing, implementation must not begin.

---

# Required Outputs

Every completed implementation must provide:

- Production-ready code
- Appropriate tests
- Updated documentation
- Notes describing any implementation decisions

---

# Engineering Responsibilities

The engineer shall:

- Follow the approved specification.
- Follow Argus coding standards.
- Preserve architectural simplicity.
- Write readable, maintainable code.
- Document meaningful implementation decisions.
- Report ambiguities before implementing assumptions.

---

# Engineering Prohibitions

The engineer shall not:

- Invent undocumented requirements.
- Change public interfaces without approval.
- Modify architecture without review.
- Ignore failing tests.
- Introduce unnecessary dependencies.
- Sacrifice readability for cleverness.

---

# Handling Ambiguity

If the specification is unclear:

1. Stop implementation.
2. Identify the ambiguity.
3. Propose one or more reasonable options.
4. Request clarification.

Do not guess.

---

# Definition of Done

Implementation is complete only when:

- Specification has been fully implemented.
- Tests pass.
- Documentation is updated.
- Code follows Factory standards.
- No known issues remain undocumented.

---

# Guiding Principle

The engineer's responsibility is not to write the most code.

The engineer's responsibility is to produce the simplest correct implementation of the approved design.
